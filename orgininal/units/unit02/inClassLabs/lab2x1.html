NOTE:  In the MATC classrooms and labs we will be using VMware Workstation 6.5 or 7 to run our Linux virtual machines (VMs). However our Linux VMs are set up to also be used with VMware Player (for Windows), VMware Fusion (for Apple OSX), and VMware Workstation 6.5 or 7 for Linux.

VMware Player may be downloaded free from the VMWare site. As an MATC student, you can get a one year license for the other VMware products. Information about downloading VMware products may be obtained from the lab manager in room 220B, and also may have been given to you by your instructor in the first class session.

  1. Exercise 1

    1. Copy the Linux virtual machine (VM) from the DVD given to you by your instructor to your lab machine or your external USB drive. The VM is in a folder or directory named something similar to Ubuntu9_w_PHP on the CD. Drag the compressed folder to a folder (directory) on either your C: drive or your USB drive. You might want to create a folder on your USB drive for all virtual machines that you use.

    2. Start up VMWare Workstation on your lab machine by double-clicking on the desktop icon.

    3. At the VMWare Console, select File=>Open . In the dialog box select Browse , and locate the Ubuntu VM on your external drive. Select the file "Ubuntu.vmx" in the Ubuntu9_w_PHP directory.

    4. Start up the Ubuntu VM by clicking on the green arrow next to the word Start . A dialog box will appear asking whether you want to create a new unique ID number (UUID) for your VM. Set the radio button to Create and click OK .

      NOTE: When you copy or move a VM to a different location, VMWare "notices" and asks this question. The purpose is to assure that each VM has a unique internal ID. Using multiple VMs with the same internal ID may cause a malfunction, potentially corrupting a VM!

    5. The Ubuntu VM will take some time to start up. You may get a dialog box asking whether the VM should check for the existence of a floppy drive. We recommend you answer Yes, so a floppy drive will be used if available on the host.

    6. When the GUI login screen appears, use the following user name and password to log in:

      User:             student
      Password:    student

  2. Exercise 2

    1. Open either the jEdit or the BlueFish editor by double clicking the icon on your Linux desktop. In the editor, navigate to the directory /var/www/labs .

      NOTE: If the labs directory does not exist, try to create it in the editor. If this fails, ask your instructor for help. Linux will only let you (as user student) create a file or directory in a directory owned by user student .

    2. Create a PHP script named hello.php containing the following text:

      <html>
       <head>
        <title>PHP Hello World!</title>
       </head>
       <body>
        <?php echo 'Look out world, here is another PHP script!' ?>
       </body>
      </html>

      and save it into your labs directory.

    3. Create a second PHP script named phpinfo.php containing the following text:

      <html>
       <head>
        <title>phpinfo()</title>
       </head>
       <body>
        <?php phpinfo() ?>
       </body>
      </html>

      and save it into your labs directory.

    4. Test your two scripts by opening Firefox and loading the following URLs:

      http://localhost/labs/hello.php

      http://localhost/labs/phpinfo.php

    5. Note that PHP scripts must be placed in a directory under the web server's document root and accessed through the document root for the PHP code to be executed. What happens if you load your hello.php script using the following URL?

      file://var/www/labs/hello.php

    6. Show your scripts and the results of loading the above URLs to your instructor.

  3. Exercise 3

    1. When done showing your code to the instructor, close jEdit or BlueFish, Firefox and any other applications you have open.

    2. Log out of the student account (by selecting System=>Quit and then Shutdown to shut down the Ubuntu VM, returning to the VMWare console.

    3. The Ubuntu VM will automatically "power down" after completing the Shutdown process. It is still important to WAIT until you see the VMWare configuration screen. Until that screen re-appears, VMWare is flushing the disk buffers to your hard disk, and that can take a minute or two. DO NOT interrupt that process.

    4. THIS IS VERY IMPORTANT: Do not ever just detach your USB drive, without first using the Windows "Safely Remove Hardware" system tray icon, or dismounting the USB drive on Linux or Apple OSX. This will corrupt your VM, causing you to lose all PHP scripts and other data saved on it!
